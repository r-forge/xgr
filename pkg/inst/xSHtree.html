xSHtreeR Documentation

Function to draw multiple component planes reorded within a tree using ggplot2

Description

xSHtree is supposed to draw multiple component planes reorded within a tree using ggplot2.

Usage

xSHtree(sMap, phylo = NULL, layout = c("rectangular", "fan",
"partition"), colormap = "spectral", zlim = NULL, tip.size = 10,
width = 1)

Arguments

sMap

an object of class "sMap"

phylo

an object of class "phylo" or NULL

layout

the visual layout. It can be "rectangular" or circular stype ("fan" and "partition")

colormap

short name for the colormap. It can be one of "jet" (jet colormap), "bwr" (blue-white-red colormap), "gbr" (green-black-red colormap), "wyr" (white-yellow-red colormap), "br" (black-red colormap), "yr" (yellow-red colormap), "wb" (white-black colormap), and "rainbow" (rainbow colormap, that is, red-yellow-green-cyan-blue-magenta). Alternatively, any hyphen-separated HTML color names, e.g. "blue-black-yellow", "royalblue-white-sandybrown", "darkgreen-white-darkviolet". A list of standard color names can be found in http://html-color-codes.info/color-names

zlim

the minimum and maximum z values for which colors should be plotted, defaulting to the range of the finite values of displayed codebook matrix

tip.size

the tip/title size

width

the width/height of embedded ggplot obejcts

Value

a ggplot2 object

Note

none

See Also

xSHtree

Examples

## Not run: 
library(XGR)

gp <- xSHtree(sMap, layout="rectangular", width=1)
gp
ls_gp <- gp$ls_gp

# advanced use
tree <- visTreeBootstrap(t(sMap$codebook))
gp <- xSHtree(sMap, tree, layout="rectangular", width=1)
gp <- xSHtree(sMap, tree, layout="fan", width=0.4)
gp <- xSHtree(sMap, tree, layout="partition", width=0.7)
